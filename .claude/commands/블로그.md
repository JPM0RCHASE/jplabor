# 스포티파이 형태로 블로그 자료 만들기

홈페이지의 JP 칼럼/가이드북/사례 내용을 네이버 블로그용 자료로 변환한다.

## 발동 트리거
다음 중 하나면 이 스킬을 실행한다:
- `/블로그 [칼럼 제목]`
- "스포티파이 형태로 블로그 자료 만들어줘 — [칼럼 제목]"
- "[제목] 블로그 자료(자료) 만들어줘"

입력으로 받은 칼럼 제목/내용: $ARGUMENTS

## 생성 구조 (3종 + 해시태그)
1. **썸네일** (1200×630) → **이미지(.png)** — 스포티파이 다크(#121212, #1ed760)
2. **본문** → **텍스트(.txt)** — 검색 색인 위해 절대 이미지로 만들지 않음
3. **면책조항** → **이미지(.png)** — 밝은 톤, 글 하단용
4. **해시태그 10개** → 본문 .txt 맨 끝에 포함

⚠️ 본문을 이미지로 만들면 네이버 검색에 안 잡힌다. 반드시 텍스트.

## 실행 절차
1. 홈페이지 `index.html`에서 해당 칼럼/가이드 본문을 읽어온다.
2. `blog-assets/blog-[slug].html` 생성 (#thumb / #disc 두 섹션만 — 캡처 대상).
   - 썸네일: 제목에서 핵심 단어 1개를 `.acc`(초록)로 강조, 상단 라벨 "JP LABOR · 노동법 실무", 하단 워터마크 "jplabor.com"
   - 면책: ⚖️ 아이콘 + "본 블로그에 게재된 내용은 일반적인 정보 제공을 목적으로…" 고정 문구
   - [slug] = **자료의 핵심 주제**로 명명 (예: 인사팀 노조가입, 5인미만적용, 통상임금).
3. 이미지 2장 캡처 → 폴더 `blog-output/YYMMDD_[slug]/`에 자동 저장 (날짜 접두어 자동):
   ```bash
   node blog-assets/shot-blog.mjs blog-assets/blog-[slug].html
   ```
   (집 PC는 그대로, 클라우드 환경은 `PW_CHROME=/opt/pw-browsers/.../chrome` 앞에 붙임)
4. `blog-assets/blog-output/YYMMDD_[slug]/본문.txt`를 아래 기준으로 작성:

   ### 제목 규칙
   - **🔑 검색 키워드를 제목 앞 15자 안에 배치 (최우선)**: 독자가 네이버에 실제로 치는 단어(예: "특별연장근로 인가 신청", "5인 미만 해고")를 제목 맨 앞에 둔다. 후킹 문구는 그 뒤에. (C-Rank·검색 노출 직결 — 키워드가 뒤로 밀리면 유입이 죽는다)
     · ❌ `주 52시간 넘기면 무조건 불법? "특별연장근로 인가" 받으면 합법입니다`
     · ✅ `특별연장근로 인가 신청방법 총정리 — 주 52시간 합법 초과 (2026)`
   - 클릭 유도형 + 혜택 중심; 법률 용어 대신 대중 "돈 키워드" 병행
   - 연도·금액·'하는 법'·'신청방법' 등 구체성 요소 포함
   - **본문.txt 맨 첫 줄에 제목 후보 3개를 먼저 제시** (3개 모두 핵심 키워드를 앞쪽에 둔 채, 아래 3가지 유형으로):
     · 공포/리스크형: "이것 모르면 벌금·고소" 등 처벌 강조
     · 스토리/공감형: 실제 상황처럼 묘사해 클릭 유도
     · 실용/정보형: "총정리", "완벽 정리", "한 번에" 등 정보량 강조
   - 형식: `📌 제목 후보` 블록으로 구분, 본문 시작 전에 배치

   ### 본문 구성 (모바일 최적화 · AdPost 수익 최적화 · Naver C-Rank 권위)
   - **🎯 타깃 독자 1명 고정**: 이 블로그의 독자는 **사업주·인사담당자**다. 근로자 관점("이렇게 받아내세요")이 아니라 사용자 관점("이렇게 대응/예방하세요")으로 일관되게 작성. 한 글 안에서 타깃이 흔들리지 않게 한다.
   - **3줄 후킹 인트로**: 공감 → 독자가 잃을 수 있는 돈/권리 → 해결 약속 (각 1줄씩)
   - **번호 소제목**: `▌01.` 형식으로 섹션 구분
   - **단락 길이**: 3~4줄마다 줄 바꿈 (모바일 가독성)
   - **강조**: 핵심 용어/결론은 **굵게**, 형광펜 필요 부분은 `[형광펜]` 표시
   - **이미지 포인트**: `[이미지: 설명]` 형태로 2~4개 표시 (블로그 업로드 시 해당 위치에 이미지 삽입)
   - **분량**: 공백 포함 약 1,500~2,500자
   - **계산 예시 필수**: 실명 페르소나(예: "김철수 씨, 월급 250만 원…") + 구체 숫자로 버릿 또는 텍스트 표 형태 1개 이상
   - **Q&A 섹션 필수**: 자주 묻는 실전 질문 2~3개를 대화체로
   - **핵심 요약**: 짧은 테이크어웨이 정리
   - **마무리 CTA (2단 구조 — 수익 전환 필수)**:
     · ① 초저허들 댓글 유도: 답하기 쉬운 한 줄 질문 (예: "우리 회사도 해당되는지 궁금하면 업종만 댓글로 남겨주세요")
     · ② 상담 전환 문구: "사안마다 결론이 다르므로 정확한 검토는 전문가 상담이 필요합니다 → 공인노무사 JP 상담문의(이메일/댓글)". 노무사 블로그의 본체 수익은 광고가 아니라 **자문 계약 전환**임을 항상 의식.
   - **내부링크 (체류시간·시리즈화)**: `[내부링크: 주제]` 형태로 1~2개 관련 글 제안. 같은 카테고리 글은 시리즈처럼 서로 연결해 1명이 2~3개 글을 읽게 유도.
   - **정확성 (필수)**: 모든 요율·기준·기한·법령에 적용 연도/날짜 명시; 확인 불가 수치는 `(가정)` 표시 후 독자에게 최신 공식 수치 확인 안내; 이전 연도 수치 무단 재사용 금지
   - **톤**: 전문적이되 친근하게 — 법을 법 모르는 친구에게 설명하듯
   - 인용/근거: `▪ 라벨` 줄 + 내용
   - 경고/주의: `⚠️`
   - 목록: ` · `
   - **맨 끝: 내용에 어울리는 해시태그 정확히 10개** (`#태그` 공백 구분)
     · 구성 가이드: 핵심 법령/제도(3) + 쟁점 키워드(3) + 대상·상황(2) + 브랜드/직군(2)
     · 예) #노동조합 #노조법 #소극적요건 #이익대표자 #부당노동행위 #단체교섭 #노사관계 #인사노무 #공인노무사 #노무상담
5. 산출물(이미지 2 + 텍스트 1)을 사용자에게 보여주고, 변경 시 git add -f 로 png 포함해 커밋·푸시.

## 출력 파일 (날짜_주제 폴더로 정리)
```
blog-assets/blog-output/YYMMDD_[핵심주제]/      예: 260609_인사팀 노조가입/
  thumbnail.png    ← 네이버 대표 이미지
  본문.txt          ← 본문 텍스트 + 해시태그 10개 (복붙용)
  disclaimer.png   ← 글 맨 아래 면책조항
```
집 PC 기준: /Users/cheonjipyeong/jplabor/blog-assets/blog-output/YYMMDD_[핵심주제]/
날짜(YYMMDD)는 shot-blog.mjs가 실행일 기준으로 자동 생성.

## 네이버 업로드 순서
1. 글쓰기 → 대표 이미지: `thumbnail.png`
2. 본문에 `본문.txt` 복붙 (해시태그 포함)
3. 맨 아래 이미지: `disclaimer.png`
4. 발행

## 📈 수익화 운영 원칙 (자료 생성 시 항상 의식할 것)
블로그 자료의 목적은 "예쁜 글"이 아니라 **트래픽 → 체류 → 상담 전환**이다. 자료를 만들 때 아래를 점검한다.

1. **검색 유입 최우선**: 디자인보다 키워드. 썸네일은 이미 충분한 수준이므로 과다 투자 금지. 제목·소제목·본문에 검색 키워드를 자연스럽게 반복 배치.
2. **타깃 단일화**: 모든 자료는 사업주·인사담당자向. 근로자向 톤이 섞이면 안 된다.
3. **수익 깔때기 의식**: 무료 정보(본문) → 애드포스트(소액) → **상담 자문(본체 수익)**. CTA에 상담 경로가 빠지면 안 된다.
4. **시리즈·내부링크로 체류 확보**: 단발 글 지양. 같은 카테고리 글끼리 `[내부링크]`로 묶어 회차감 부여.
5. **발행 꾸준함 > 단발 명작**: C-Rank는 규칙적 발행에 가점. 한 번에 몰아 만들더라도 발행은 일정 간격을 권장(사용자에게 안내).
6. **단계 인식**: 이웃·일 방문이 적은 초기에는 '수익'보다 '트래픽·이웃 확보'가 먼저. 자료 추천 시 현 단계에 맞는 우선순위를 함께 제안.
